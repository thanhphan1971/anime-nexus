Goal:

We must add: countdown, eligibility messaging, and a recap section.

Constraints:
- Keep current schedule logic.
- Monthly entry: FREE for paid users only; paid users get 1 free monthly entry per cycle.
- Free users: monthly entry is paid (tokens) OR blocked with upsell message (choose token-paid if it already exists).
- Weekly entry: keep existing rules (do not change unless already discussed).

Implement:
A) Home page “Draws” section (or banner)

  - “Enter Weekly Draw”
  - “Enter Monthly Draw”

  - Paid: “You have 1 free monthly entry this cycle” (or “Already used — next cycle in X days”)

B) Draw detail page (or modal) for each draw
- Show next draw time + countdown

  - Entered / Not entered

  - After a draw happens, store winners in DB and show a simple recap list (card won + winner display name truncated or anonymized).

C) Database changes
- Create tables:
  1) draws (id, type weekly/monthly, scheduled_at, ends_at, status, created_at)
  2) draw_entries (id, draw_id, user_id, created_at, entry_type free/paid, token_cost)
  3) draw_results (id, draw_id, user_id, card_id, created_at)
- Add indexes on draw_id and user_id where needed.
- Ensure we can query “current cycle draw” and whether user already used their free monthly entry.

D) Server/API
- Endpoints:
  - GET /api/draws/next (returns next weekly + next monthly + countdown targets)
  - POST /api/draws/:id/enter (handles eligibility + token charging if needed)
  - GET /api/draws/:id/status (user entry status)
  - GET /api/draws/:id/recap (returns results list)

E) UI acceptance criteria
- Home shows both countdowns correctly.
- Paid user sees “free monthly entry available” until they use it.
