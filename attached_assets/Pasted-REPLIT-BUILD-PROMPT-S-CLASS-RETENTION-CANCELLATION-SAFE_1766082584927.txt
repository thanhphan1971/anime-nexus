1️⃣ Cancellation-Save Screen (Retention Critical)

Trigger
User attempts to cancel an active S-Class subscription.

Flow Requirements

Intercept cancellation once only

Never block cancellation

Calm, respectful tone (no guilt, no pressure)

Screen Content

Title

Before You Go

Message

You’re an important part of AniRealm.
If something isn’t working right now, we want to make this easier.

Options (all must be shown)

Option A — Stay in S-Class (Soft Save)

Keep S-Class and receive a small thank-you bonus.

• +200 tokens (one-time only)
• No renewal or billing changes

Option B — Switch to Yearly (Price Anchor)

Save more by switching to annual billing.

• Clearly display yearly savings
• No forced upsell

Option C — Cancel Membership

Continue cancellation

• Cancellation proceeds immediately
• No further prompts

Rules

Save bonus can trigger once per account lifetime

Log save attempts

If user confirms cancellation, never show save screen again

⏳ Cancellation Timing Rule (MANDATORY)

If a user cancels S-Class:

• S-Class access remains active until the end of the current billing period
• No partial refunds are issued
• No additional charges occur
• Subscription auto-expires at period end
• User is downgraded to Free only after expiration

UI Requirements

Display: “Active until [date]” after cancellation

Clearly communicate continued access during remaining period

Do NOT revoke benefits early

State Machine

Active → Canceled (Pending Expiry) → Expired → Free
